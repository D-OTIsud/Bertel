Fonctionnalités & portée

Une carte type vers famille et un résolveur de libellés canoniques, source de vérité unique partagée par l'explorateur, l'éditeur et le tiroir. Corrige le défaut d'audit des deux taxonomies divergentes.

Comportement

  • Une seule carte canonique type vers archétype : 19 types DB regroupés en 7 familles (HEB, RES, ASC, ITI, VIS, SRV, FMA), ORG restant hors éditeur.
  • Quatre résolveurs : resolveTypeLabel, resolveArchetype, resolveRoleLabel, resolveSchemeLabel.
  • La couleur de pastille vient toujours de l'archétype (classe acc-*), jamais d'une couleur posée à la main.
  • Corrige le défaut : aujourd'hui utils/facets.ts:24-32 range LOI dans « Activités », ASC sous « Services », VIL sous « Visites », en contradiction avec archetypes.ts.

Critères d'acceptation

  • Aucun code brut affiché comme libellé nulle part (type, rôle, schéma de distinction).
  • Le groupe d'explorateur d'un type est identique à son archétype d'éditeur.
  • Une même couleur de pastille pour un type dans l'explorateur, l'éditeur et le tiroir.

Données & fichiers impactés

  • Nouveau : src/utils/labels.ts (ou extension de archetypes.ts) avec les 4 résolveurs.
  • Alignement : utils/facets.ts · recaler les buckets sur les archétypes (LOI, ASC, VIL).
  • Branchement transverse : ActiveFilterStrip.tsx, ActualisationTable.tsx, CompletenessTable.tsx, MembersTable.tsx, crm-primitives.tsx.

Notes

  • Les 7 accents --acc-* et les classes type-pill acc-* viennent de bertel-ui.css (impl. 1.1) : aucune nouvelle couleur.
  • Les libellés FR sont la couche d'affichage ; les codes DB restent inchangés en base.
