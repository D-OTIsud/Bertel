Fonctionnalités & portée

Carte type-aware

  • Couleur + icône du marqueur dérivées de l'archétype (même palette que les cartes 3.1 et les pastilles 2.1).
  • Pin de cluster teinté par composition (anneau multi-couleurs proportionnel aux types groupés), plus un compteur · fini le vert uniforme.
  • Popup au survol adaptée au type (titre + pastille de type + méta dédiée), comme la ligne méta des cartes.
  • Distinctions conservées dans la popup : le classement (étoiles pour un HEB classé, ex. 3 étoiles ; clés/épis pour les autres hébergements), la puce-label avec son logo et son point d'état (accordé / en cours / expiré) et 1 à 2 tags colorés y restent visibles, alignés sur les cartes et la fiche. Les types non hébergement (RES, ASC, ITI, VIS, FMA) n'affichent pas d'étoiles mais conservent label et tags.
  • Légende ancrée bas-gauche décodant les 7 archétypes (absente aujourd'hui).

Reprise d'erreur

  • Remplace le remplacement plein-écran (views/ExplorerPage.tsx:61-67) par un bandeau inline + « Réessayer ».
  • La dernière donnée valide reste affichée : une requête carte/références en échec ne vide plus tout l'Explorateur.
  • Le « Réessayer » relance uniquement la requête fautive (carte), sans recharger la liste.

Données & fichiers

  • src/components/explorer/MapPanel.tsx : rendu des marqueurs/clusters/popup type-aware + état d'erreur local.
  • src/config/map-markers.ts : table archétype → couleur + icône (source unique partagée avec les cartes).
  • views/ExplorerPage.tsx : retirer le remplacement plein-écran ; isoler l'erreur carte du reste.
  • CSS de cluster (anneau de composition + compteur) dans styles.css.

Critères d'acceptation

  • Marqueurs et clusters distinguent visuellement le type.
  • La légende est présente et décode les 7 archétypes.
  • La popup au survol surface le classement (étoiles pour un HEB classé), la puce-label avec son état et les tags ; pas d'étoiles sur les types non hébergement.
  • Une erreur de requête références/cartes s'affiche inline ; l'Explorateur reste utilisable.
