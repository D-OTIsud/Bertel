Fonctionnalités & portée

Comportement

  • Nouveau RestaurantDetailView dédié (aujourd'hui un quasi-clone du gabarit générique).
  • Aperçu = identité culinaire : cuisine (chips), régimes (badges), gamme de prix, statut « Ouvert ».
  • Section Carte / Menu = groupes de plats (Entrées / Plats / Desserts) avec nom · description · prix aligné à droite, + bouton « Voir la carte (PDF) ».
  • Les équipements passent en second plan, jamais en tête de fiche.
  • Classement / labels / tags conservés sur la fiche (retour PO) : section « Classements & labels » en puces-logo (état accordé · en cours · expiré) + rangée de tags colorés.
  • La miniature héros porte des cocardes label « à cheval » sur le coin (Qualité Tourisme, Tourisme & Handicap) ; aucune cocarde de classement étoiles pour un RES (réservée aux hébergements).

Critères d'acceptation

  • Cuisine + menu structuré + régimes visibles sans clic supplémentaire.
  • Les équipements ne mènent pas la fiche.
  • Onglet « Carte » présent uniquement si un menu existe (sinon masqué).
  • Classement (N/A pour RES), labels et tags restent visibles sur la fiche ; la miniature affiche les cocardes label quand au moins un label est porté, et jamais d'étoiles pour un restaurant.

Données & fichiers

  • src/features/object-drawer/ObjectDetailView.tsx : RestaurantDetailView + sections cuisine/menu/régimes.
  • Config ARCHETYPE_SECTIONS : ordre RES (aperçu culinaire · carte · horaires · médias · contact).
  • Données §104 : object_cuisine_type, menu structuré (object_menu / object_menu_item), régimes (dietary), carte PDF (object_document).
  • Gamme de prix dérivée de object.price_range ; statut via open_now (sens horaire pour RES).
  • Distinctions : seuls les labels granted sont affichés (un label requested / expired n'apparaît pas côté public), sans pastille d'état ; tags via tag_link (couleur --tag). Le classement en étoiles n'est rendu que pour les archétypes HEB.
lightbulb Le menu structuré est déjà autorable côté éditeur depuis §104 ; cette fiche est la première surface publique à le rendre. Le PDF n'apparaît que si object_document porte une carte.
