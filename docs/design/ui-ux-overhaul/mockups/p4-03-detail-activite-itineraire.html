Fonctionnalités & portée

Comportement

  • ASC : nouveau ActivityDetailView qui expose object_act · KPIs (durée · niveau · groupe · âge), point de RDV, encadrement (prestataire), équipement fourni, puis description.
  • ITI : ItineraryDetailView affiche les vraies étapes de object_iti_stage (nom + description réels), un profil altimétrique, et la trace GPX si elle existe.
  • Les deux fiches sont pilotées par config (mêmes briques entête / tabs / aside).
  • Suppression de l'interpolation « Étape n · 4,3 km » du gabarit actuel.
  • Labels & tags conservés (retour PO) : un bloc « Labels & tags » présent sur chaque fiche. Seuls les labels valides (granted) sont affichés, sans pastille d'état. Les pratiques (canyoning, randonnée, boucle, panorama, eau potable…) sont rendues comme .tag-chip colorés (couche d'affichage §09).
  • Signet « à cheval » sur la miniature : la vignette héro porte un .thumb__labels dont la pastille-logo du label (ici lbl-th) chevauche le coin de l'image. Pas de cocarde de classement ici : ASC et ITI ne sont pas classés en étoiles/clés/épis (seul l'hébergement HEB l'est).

Critères d'acceptation

  • Les faits d'activité (durée/niveau/RDV/encadrement) sont affichés.
  • Les étapes d'itinéraire sont réelles, jamais interpolées.
  • GPX présenté seulement s'il existe ; sinon omis honnêtement (pas de bouton mort).
  • Labels et tags restent visibles sur la fiche ; un label non valide (en cours / expiré) n'est pas affiché.
  • La miniature des objets labellisés porte le signet du label « à cheval » sur son coin ; aucun classement étoiles/clés/épis n'est affiché pour ASC ni ITI.

Données & fichiers

  • src/features/object-drawer/ObjectDetailView.tsx : ActivityDetailView + ItineraryDetailView branchés sur leurs vraies sources.
  • Config ARCHETYPE_SECTIONS : ordre ASC (KPIs · pratique · description) et ITI (KPIs · étapes · tracé/GPX).
  • ASC : object_act (durée, niveau, taille de groupe, âge, RDV, encadrement, équipement).
  • ITI : object_iti (distance/D+/durée/forme/difficulté) + object_iti_stage (étapes) + object_iti.geom → GPX.
