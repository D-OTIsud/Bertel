Phase 7 · Implémentation 7.1

Paramètres : hub à rail & « Mon compte »

Fondation de la refonte des Paramètres. /settings n'est plus un défilement plat de 7 cartes identiques (rayon 32px) mélangeant 3 publics : c'est une console à rail de sections groupées par périmètre, un seul panneau visible à la fois, chaque groupe affiché selon le rôle. Cette implémentation pose le shell + le groupe Mon compte (préférences de langue, carte d'état Session & rôle qui remplace le dump « Runtime » de debug). Les autres modules s'y branchent (7.2 → 7.5).

Les informations d'environnement quittent la surface principale : repliées dans « Diagnostic », libellées en clair, et plus jamais affichées comme des « fonctionnalités ».

Fonctionnalités & portée

Transforme /settings en console à rail (un panneau à la fois, groupes gated par rôle) et livre le groupe « Mon compte ». Corrige les lacunes IA, jargon/debug, accents et échelle de rayon de l'audit.

Comportement

  • Rail de 3 groupes : Mon compte (tout le monde) · Mon organisation (admin ORG) · Plateforme (super-admin). Un seul panneau rendu à la fois (divulgation progressive).
  • Un groupe n'apparaît que si le rôle le permet : un lecteur ne voit que « Mon compte » · fin du mur de ~60 contrôles désactivés.
  • Préférences : langue d'interface (FR/EN/DE), au moins une active, persistée sur app_user_profile.lang_prefs.
  • Session & rôle : carte d'état lisible (rôle en libellé FR, organisation, état de session, marque). Le contenu env/debug est replié dans « Diagnostic ».
  • Rayon ramené à l'échelle --r-md/--r-sm (fini le 32px) ; surfaces plates ; :focus-visible sur le rail, chips, résumés details.
  • « Paramètres » accentué partout (sidebar, topbar, drawer) · retrait du libellé « Settings ».

État (rôles)

  • Lecteur / éditeur : voit « Mon compte » uniquement.
  • Admin d'ORG (rang ≥ 10) : ajoute « Mon organisation ».
  • Super-admin : ajoute « Plateforme ».

Données & fichiers

  • src/views/SettingsPage.tsx : passage carte-pile → shell rail + routeur de panneaux (état d'onglet en URL : /settings?section=preferences).
  • Nouveaux features/settings/SettingsRail.tsx + panes/AccountPreferences.tsx + panes/SessionRolePane.tsx.
  • Langue : updateCurrentUserProfile({ lang_prefs }) (existant).
  • Sidebar.tsx / TopBar.tsx / ProfileDrawer.tsx : libellé « Paramètres » accentué unique.
  • Panneau « Runtime » supprimé ; env migré vers le details Diagnostic.

Critères d'acceptation

  • Un seul panneau visible ; navigation au rail reflétée dans l'URL ; retour navigateur fonctionne.
  • Groupes rendus selon le rôle ; aucun mur de contrôles désactivés pour un lecteur.
  • Aucune chaîne de debug runtime sur la surface principale ; tous les libellés FR accentués.
  • Rayon = échelle de tokens ; :focus-visible visible au clavier sur rail/chips/détails.
  • La préférence de langue persiste et se recharge.
