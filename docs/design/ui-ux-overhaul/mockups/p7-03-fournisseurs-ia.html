Fonctionnalités & portée

Porte AiProviderSettings des objets style={{}} en ligne et des classes hors-scope (.btn / .pill-mini) vers le vocabulaire maison, et la branche comme panneau « Fournisseurs IA » du hub à rail (7.1).

Comportement

  • Lister / créer-modifier (upsert) / activer (setActive) / supprimer les fournisseurs IA.
  • Clé API en écriture seule : jamais relue ni réaffichée ; champ vide = on conserve la clé existante.
  • « Tester la connexion » vérifie le fournisseur actif via une requête serveur.
  • Les actions destructives Supprimer et Activer passent par le ConfirmDialog maison (bouton .btn--danger pour la suppression).
  • Un seul fournisseur actif à la fois ; état rendu via .badge--ok + .dot--ok.

Données & fichiers

  • src/features/settings/AiProviderSettings.tsx : porté de style={{}} + .btn/.pill-mini vers .panel / .btn / .badge / .table maison.
  • services/ai-provider : listAiProviders / upsertAiProvider / setActiveAiProvider / deleteAiProvider / testAiConnection (inchangés).
  • POST /api/admin/ai-config/test : vérifie is_platform_superuser côté serveur.
  • Rendu comme panneau « Fournisseurs IA » dans le rail du hub des Paramètres (7.1).

Critères d'acceptation

  • Plus aucun bouton natif nu nulle part : un seul vocabulaire visuel.
  • La clé API n'est jamais affichée ; le champ vide conserve la clé.
  • ConfirmDialog sur Supprimer et sur Activer (changement de fournisseur en service).
  • :focus-visible visible au clavier sur tous les contrôles (table, boutons, champs, select).
  • Gating super-admin appliqué côté serveur, pas seulement masqué dans l'UI.
  • Tous les libellés FR correctement accentués (« Clé API », « Tester la connexion », « chiffrée », « Modèle »).
